Gotcha—you don’t need “every phone,” you need smart breakpoints that cover all phone widths. Here’s a practical, device-agnostic set you can drop in today.

Recommended phone breakpoints (CSS px, portrait)
	•	≤320px — very small phones (legacy/mini)
	•	321–360px — common Android small
	•	361–390px — most modern iPhones & many Androids
	•	391–414px — large/“Plus” era phones
	•	415–480px — extra-wide phones & small foldables (folded)
	•	481–600px — big phones / phablets (also good for landscape phone tweaks)

Vanilla CSS media queries (mobile-first)

/* Base = tiny phones (≤320) */

@media (min-width: 321px) { /* small phones */ }
@media (min-width: 361px) { /* modern phones */ }
@media (min-width: 391px) { /* large phones */ }
@media (min-width: 415px) { /* XL phones */ }
@media (min-width: 481px) { /* phablets / landscape phones */ }
@media (min-width: 601px) { /* small tablets / unfolded */ }

Tailwind config (adds “phone bands” without fighting your desktop breakpoints)

// tailwind.config.js
module.exports = {
  theme: {
    screens: {
      // phone bands
      xs:  '320px',   // very small
      sp:  '360px',   // small phone
      mp:  '390px',   // modern phone
      lp:  '414px',   // large phone
      xp:  '480px',   // extra-wide phone
      // your existing app-wide breakpoints can continue here:
      sm:  '640px',
      md:  '768px',
      lg:  '1024px',
      xl:  '1280px',
      '2xl': '1536px',
    }
  }
}

Use like: class="text-base mp:text-lg lp:text-xl"

⸻

Extra tips so it “just works” on phones
	•	Design by width bands, not models. There are dozens of DPIs and device pixels; CSS px widths above are what matter for layout.
	•	Use dynamic viewport units for full-height sections:

.screen-block { min-height: 100dvh; } /* avoids the mobile URL bar jump */


	•	Respect notches/safe areas (iOS):

.safe {
  padding-top: env(safe-area-inset-top);
  padding-right: env(safe-area-inset-right);
  padding-bottom: env(safe-area-inset-bottom);
  padding-left: env(safe-area-inset-left);
}


	•	Fluid type/spacing:

html { font-size: clamp(14px, 2.5vw, 18px); }


	•	Touch targets: keep tappables ≥44×44 CSS px.
	•	Test DPR & orientation: in DevTools simulate different DPRs and rotate—fix anything that breaks at mp (390px) and lp (414px) first; that covers the majority.
